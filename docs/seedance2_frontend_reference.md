# Seedance 2.0 前端功能参考手册

> 基于火山方舟官方文档提炼，供前端开发抽象功能模块使用。
> 来源：Seedance 2.0 系列教程、提示词指南、视频生成教程（3份PDF）

## 1. 模型选型

| 模型 | Model ID | 定位 |
|------|----------|------|
| Seedance 2.0 | `doubao-seedance-2-0-260128` | 最高品质 |
| Seedance 2.0 Fast | `doubao-seedance-2-0-fast-260128` | 成本优先、速度更快 |

两者能力完全一致，区别在于品质/速度/成本权衡。

## 2. 生成模式（前端需抽象的核心能力）

### 2.1 文生视频（T2V）
- **输入**：纯文本提示词
- **前端控件**：文本输入框
- **API content**：`[{type: "text", text: "..."}]`

### 2.2 图生视频 - 首帧（I2V First Frame）
- **输入**：1张图片 + 可选文本提示词
- **前端控件**：图片上传（首帧）、文本输入框
- **API content**：
  ```json
  [
    {"type": "text", "text": "..."},
    {"type": "image_url", "image_url": {"url": "..."}}
  ]
  ```
- **注意**：不传 role 即为首帧模式；也可显式设 `role: "first_frame"`

### 2.3 图生视频 - 首尾帧（I2V First+Last Frame）
- **输入**：首帧图片 + 尾帧图片 + 可选文本提示词
- **前端控件**：首帧图片上传、尾帧图片上传、文本输入框
- **API content**：
  ```json
  [
    {"type": "text", "text": "..."},
    {"type": "image_url", "image_url": {"url": "..."}, "role": "first_frame"},
    {"type": "image_url", "image_url": {"url": "..."}, "role": "last_frame"}
  ]
  ```
- **约束**：必须恰好2张图片，分别标记 first_frame 和 last_frame

### 2.4 多模态参考生视频（R2V）【Seedance 2.0 专属】
- **输入**：文本 + 图片(0~9张) + 视频(0~3个) + 音频(0~3个)，任意组合
- **前端控件**：文本输入、多图上传、视频上传、音频上传
- **API content role 值**：
  - 图片：`reference_image`
  - 视频：`reference_video`
  - 音频：`reference_audio`
- **不支持的组合**：纯文本+音频、纯音频
- **提示词引用素材**：按同类素材在 content 数组中的顺序，用 `图片1`、`图片2`、`视频1`、`音频1` 等指代

### 2.5 编辑视频【Seedance 2.0 专属】
- **输入**：待编辑视频 + 可选参考图片/音频 + 文本描述编辑意图
- **前端控件**：视频上传、图片上传（可选）、文本输入
- **能力**：替换主体、增删改元素、局部重绘/修复
- **API content**：视频用 `role: "reference_video"`

### 2.6 延长视频【Seedance 2.0 专属】
- **输入**：1~3段视频 + 文本描述延长方向
- **前端控件**：多视频上传（最多3个）、文本输入
- **能力**：向前/向后延长，或多段视频串联补齐过渡
- **约束**：所有视频总时长 ≤ 15秒

### 2.7 生成有声视频
- **参数**：`generate_audio: true`
- **适用**：Seedance 2.0 / 2.0 Fast / 1.5 pro
- **前端控件**：开关 toggle

## 3. 输出规格参数（前端必设项）

### 3.1 分辨率（resolution）

| 值 | Seedance 2.0 | Seedance 2.0 Fast |
|----|:---:|:---:|
| `480p` | ✅ | ✅ |
| `720p` | ✅ | ✅ |
| `1080p` | ✅ | ❌ |

- **默认**：720p（如不传，服务端默认）
- **前端控件**：下拉选择（Fast 模型需禁用 1080p 选项）

### 3.2 宽高比（ratio）

| 值 | 说明 |
|----|------|
| `16:9` | 横屏（默认） |
| `4:3` | 传统横屏 |
| `1:1` | 正方形 |
| `3:4` | 传统竖屏 |
| `9:16` | 竖屏短视频 |
| `21:9` | 超宽银幕 |
| `adaptive` | 自适应（根据输入图片自动匹配） |

- **前端控件**：下拉/图标选择器
- **各分辨率下的具体像素值**（以 Seedance 2.0 为例）：

| 比例 | 480p | 720p | 1080p |
|------|------|------|-------|
| 16:9 | 864×496 | 1280×720 | 1920×1080 |
| 4:3 | 752×560 | 1112×834 | 1664×1248 |
| 1:1 | 640×640 | 960×960 | 1440×1440 |
| 3:4 | 560×752 | 834×1112 | 1248×1664 |
| 9:16 | 496×864 | 720×1280 | 1080×1920 |
| 21:9 | 992×432 | 1470×630 | 2206×946 |

### 3.3 时长（duration）

| 模型 | 范围 |
|------|------|
| Seedance 2.0 / 2.0 Fast | 4~15 秒 |
| Seedance 1.5 pro | 4~12 秒 |
| Seedance 1.0 pro / pro-fast | 2~12 秒 |

- **前端控件**：滑块或数字输入，带模型联动的 min/max 约束
- **也可用 frames 代替**：`frames` 范围 [29, 289]，满足 `25 + 4n` 格式（n为正整数）
- **duration 和 frames 二选一**

### 3.4 其他输出参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `seed` | int | 种子值，控制随机性，复用可获得相似结果 |
| `camera_fixed` | bool | 是否固定摄像头（不运镜） |
| `watermark` | bool | 是否包含水印 |
| `return_last_frame` | bool | 返回视频尾帧图（用于连续视频生成） |
| `generate_audio` | bool | 是否生成有声视频 |
| `service_tier` | `"default"` / `"flex"` | flex=离线推理（价格50%，仅1.5 pro及以下） |
| `draft` | bool | 样片模式（仅1.5 pro，480p） |
| `callback_url` | string | Webhook 回调地址 |
| `tools` | array | 工具配置，如 `[{"type": "web_search"}]` 启用联网搜索 |

## 4. 镜头/运镜描述（提示词中的关键元素）

提示词中的运镜指令通过自然语言描述，模型自动理解和执行。前端可提供预设运镜模板供用户选择：

| 运镜类型 | 中文描述 | 英文对应 |
|----------|----------|----------|
| 推镜头 | 镜头缓缓推近 / 镜头推近至特写 | Dolly in / Push in |
| 拉镜头 | 镜头缓缓拉出 / 镜头拉远 | Dolly out / Pull out |
| 环绕运镜 | 360度环绕运镜 / 镜头围绕主体旋转 | Orbit shot |
| 横摇 | 镜头向右横摇 / 镜头缓慢平移 | Pan left/right |
| 俯冲 | 第一视角俯冲 | Dive shot |
| 固定机位 | 固定机位，近景镜头 | Static shot |
| 跟拍 | 镜头跟拍手部动作 | Tracking shot |
| 特写 | 微距镜头 / 特写人物面部 | Close-up / Macro |
| 远景 | 镜头拉远展现全景 | Wide shot |

**前端建议**：提供"运镜"下拉/标签选择器，将用户选择映射为提示词片段注入 prompt。

## 5. 参考素材引用规则（前端提示词构建关键）

### 5.1 素材指代语法
- 提示词中使用 `图片1`、`图片2`、`视频1`、`音频1` 等格式引用素材
- **序号 = content 数组中同类素材的出现顺序**（从1开始）
- **不支持** 用 Asset ID 指代素材

### 5.2 图片参考模板
```
参考/提取/结合 图片n 中的 [主体描述]，生成 [画面描述]，保持 [特征] 一致。
```

### 5.3 视频参考 - 动作
```
参考 视频n 的 [动作描述]，生成 [画面描述]，保持动作细节一致。
```

### 5.4 视频参考 - 运镜
```
参考 视频n 的 [运镜描述]，生成 [画面描述]，保持运镜一致。
```

### 5.5 视频参考 - 特效
```
参考 视频n 的 [特效描述]，生成 [画面描述]，保持特效一致。
```

### 5.6 音频参考
- 音色参考：`角色说："台词"，音色参考 音频n。`
- 音频内容参考：`[出现时机] 音频n。`

### 5.7 文字生成
- 广告语：`「文字内容」+「出现时机」+「出现位置」+「出现方式」，「文字特征」`
- 字幕：`画面底部出现字幕，字幕内容为"……"，字幕需与音频节奏完全同步。`
- 气泡台词：`角色说："……"，角色说话时周围出现气泡，气泡里写着台词。`

## 6. 提示词构建公式

### 6.1 基础公式
```
提示词 = 主体 + 动作 + 背景/场景 + 光影/风格 + 镜头/运镜 + 音效/氛围
```

### 6.2 时间轴分段描述（高级用法）
```
首帧为图片1，[0-2秒描述]；2-4秒：[描述]；4-6秒：[描述]；尾帧定格为图片2。
```

### 6.3 多模态参考提示词模板
```
全程使用视频1的 [构图/运镜方式]，全程使用音频1作为背景音乐。
[画面描述]，首帧为图片1，[动作描述]，尾帧定格为图片2。
背景声音统一为 [音色描述]。
```

### 6.4 视频编辑提示词模板
- 增加元素：`在 视频n 的 [时间位置] + [空间位置]，增加 [元素描述]。`
- 删除元素：`删除 视频n 中的 [元素]，视频其他内容保持不变。`
- 修改元素：`将 视频n 中的 [旧元素]，替换为 [新元素]。`

### 6.5 延长视频提示词模板
- 向后延长：`生成 视频1 之后的内容，[描述]。`
- 向前延长：`向前延长 视频1，[描述]。`
- 轨道补齐：`视频1 + [过渡描述] + 接 视频2 + [过渡描述] + 接 视频3`

## 7. 输入素材格式要求

### 7.1 图片
| 项目 | 要求 |
|------|------|
| 传入方式 | URL / Base64 / asset:// ID |
| 格式 | jpeg, png, webp, bmp, tiff, gif, heic, heif |
| 单张大小 | < 30 MB |
| 宽高比范围 | (0.4, 2.5) |
| 宽高长度 | (300, 6000) px |
| 数量限制 | 首帧:1张 / 首尾帧:2张 / 多模态参考:1~9张 |

### 7.2 视频
| 项目 | 要求 |
|------|------|
| 传入方式 | URL / asset:// ID |
| 格式 | mp4, mov（编码 H.264/H.265，音频 AAC/MP3） |
| 分辨率 | 480p, 720p, 1080p |
| 单个时长 | [2, 15] 秒 |
| 总时长 | 所有视频 ≤ 15 秒 |
| 单个大小 | < 50 MB |
| 帧率 | [24, 60] FPS |
| 数量限制 | 最多 3 个 |

### 7.3 音频
| 项目 | 要求 |
|------|------|
| 传入方式 | URL / Base64 / asset:// ID |
| 格式 | wav, mp3 |
| 单个时长 | [2, 15] 秒 |
| 总时长 | 所有音频 ≤ 15 秒 |
| 单个大小 | < 15 MB |
| 数量限制 | 最多 3 段 |

### 7.4 请求体限制
- 请求体总大小 ≤ 64 MB
- 大文件不要用 Base64，用 URL 传入

## 8. API 调用流程

### 8.1 异步任务流程
```
1. POST /contents/generations/tasks  →  返回 task_id
2. GET  /contents/generations/tasks/{id}  →  轮询状态
3. 状态 = succeeded  →  content.video_url 取视频
```

### 8.2 任务状态
| 状态 | 含义 |
|------|------|
| `queued` | 排队中 |
| `running` | 生成中 |
| `succeeded` | 已完成 |
| `failed` | 失败 |
| `expired` | 已过期 |

### 8.3 关键时间限制
- 视频 URL 有效期：**24小时**（务必及时下载/转存）
- Draft 视频 ID 有效期：**7天**

## 9. 联网搜索工具【New】

- 参数：`tools: [{"type": "web_search"}]`
- 仅适用于**纯文本输入**
- 模型自主判断是否搜索（商品、天气等时效性内容）
- 会增加一定时延

## 10. 真人肖像限制与解决方案

Seedance 2.0 **不支持直接上传含真人人脸的参考图/视频**。解决方案：

| 方案 | 说明 |
|------|------|
| 信任模型产物 | 同账号下 Seedance 2.0/2.0 Fast 近30天内生成的含人脸视频可作为输入二次创作 |
| 预置虚拟人像库 | 平台提供免费合规虚拟人像，用 `asset://<asset-id>` 传入 |
| 已授权真人素材 | 通过真人认证+授权后上传，获得 asset ID |

## 11. 前端功能模块建议

基于以上文档，前端可抽象为以下功能模块：

### 11.1 模式选择器
- 文生视频 / 图生视频（首帧）/ 图生视频（首尾帧）/ 多模态参考 / 编辑视频 / 延长视频
- 不同模式显示不同的输入区域

### 11.2 素材上传区
- 根据模式动态显示：首帧图片、尾帧图片、参考图片(多张)、参考视频(多个)、参考音频(多个)
- 格式校验、大小校验、数量上限校验
- 素材排序（影响提示词中的 `图片1`、`图片2` 引用）

### 11.3 提示词编辑器
- 文本输入框（必填/选填视模式而定）
- 可选：预设运镜模板注入、时间轴分段编辑、素材引用快捷插入

### 11.4 输出规格面板
- 分辨率下拉（联动模型：Fast 禁用 1080p）
- 宽高比选择（图标/下拉）
- 时长滑块（联动模型范围）
- 生成有声视频开关
- 水印开关
- 固定摄像头开关
- 种子值输入（高级选项）

### 11.5 高级选项（可折叠）
- 联网搜索开关（仅纯文本模式可用）
- 离线推理模式（仅非2.0模型）
- 样片模式（仅1.5 pro）
- Webhook 回调地址
- 尾帧返回开关

### 11.6 任务管理
- 创建任务 → 轮询状态 → 展示结果
- 任务列表（历史记录）
- 视频下载/预览
- 尾帧图展示（如开启 return_last_frame）

## 13. 常见坑 & 注意事项

1. **首尾帧画面跳变**：输入图片与输出视频分辨率宽高比不一致时会出现。解决方案：裁剪图片匹配目标比例，或设 ratio 为 `adaptive`。
2. **素材引用序号**：必须按 content 数组中同类素材的顺序，从1开始计数。不支持 Asset ID 指代。
3. **不支持纯文本+音频**：音频必须搭配图片或视频。
4. **URL 有效期**：生成的视频 URL 仅 24 小时有效，需及时转存。
5. **人脸限制**：Seedance 2.0 系列不支持直接上传含真人人脸的素材。
6. **请求体大小**：总大小 ≤ 64 MB，大文件用 URL 传入。
7. **并发限制**：企业用户最大并发 10，个人用户最大并发 3。
8. **RPM 限制**：企业 600/分钟，个人 180/分钟。
